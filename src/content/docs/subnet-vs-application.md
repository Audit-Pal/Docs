# Subnet vs Application

## Innovation Hub vs Production Environment

The AuditPal ecosystem has two distinct environments that serve different purposes:

## Comparison

| Aspect | Bittensor Subnet | AuditPal Application |
|--------|-----------------|---------------------|
| **Purpose** | R&D and competitive evolution | Production-grade audit delivery |
| **Users** | Miners, validators, researchers | Auditors, developers, teams |
| **Environment** | Decentralized, competitive | Centralized UX, stable |
| **Output** | Raw findings + scores | Polished audit reports |
| **Quality Gate** | Benchmark evaluation | Curated best results |
| **Access** | Open to any miner | Authenticated users |

## How They Connect

```
Subnet (Innovation)              Application (Production)
┌─────────────────┐              ┌─────────────────────┐
│ Miners compete   │──────────►  │ Best results curated │
│ on benchmarks    │   Quality   │ for end users        │
│                  │   Gate      │                      │
│ New approaches   │◄────────── │ Real-world contracts │
│ evolve here      │  Feedback   │ tested here          │
└─────────────────┘              └─────────────────────┘
```

## The Flywheel

1. **Subnet produces innovation** — Diverse AI approaches compete and evolve
2. **Application curates quality** — Only the best findings reach end users
3. **Real-world usage generates feedback** — New contract patterns feed back to improve benchmarks
4. **Better benchmarks drive better miners** — The cycle continues

> The subnet is the engine. The application is the car. You don't need to understand the engine to drive.
